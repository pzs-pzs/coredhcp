# RFC 9915 - DHCPv6 协议中文摘要

## 📋 基本信息

- **标题**: IPv6 动态主机配置协议 (DHCPv6)
- **状态**: 互联网标准 (STD 102)
- **发布时间**: 2026 年 1 月
- **替代**: RFC 8415
- **作者**: Tomek Mrugalski 等 5 人

---

## 🎯 核心功能

DHCPv6 是一个客户端/服务器协议，用于：
1. **分配 IPv6 地址** 给设备
2. **委派 IPv6 前缀** (用于路由器)
3. **提供其他配置参数** (DNS、NTP 等)
4. 支持 **无状态** 和 **有状态** 两种模式

---

## 🔄 相比 RFC 8415 的主要变更

### 移除的功能 (已废弃)
| 功能 | 原因 |
|------|------|
| **IA_TA (临时地址)** | 未被广泛实现，可用 IA_NA 替代 |
| **Server Unicast 选项** | 实际很少用，中继代理需要通信 |
| **UseMulticast 状态码** | 客户端始终使用组播 |

### 其他改进
- 整合了已验证的勘误报告
- 明确了客户端/服务器/中继代理使用的 **UDP 端口**
- 增加了隐私和安全考虑

---

## 📡 工作模式

### 1. 无状态 DHCP (Stateless)
- 只分发配置参数，**不分配地址**
- 地址通过 SLAAC 自动生成
- 服务器无需跟踪状态

### 2. 有状态 DHCP (Stateful)
- 分配 IPv6 地址和/或前缀
- 服务器维护租约状态
- 支持地址续租、释放等操作

### 3. 前缀委派 (PD)
- 用于 CPE 路由器
- 自动委派 IPv6 前缀给下游网络

---

## 💬 消息类型

| 消息 | 方向 | 用途 |
|------|------|------|
| **Solicit** | 客户端→服务器 | 发现服务器 |
| **Advertise** | 服务器→客户端 | 服务器响应 |
| **Request** | 客户端→服务器 | 请求配置 |
| **Reply** | 服务器→客户端 | 返回配置 |
| **Renew** | 客户端→服务器 | 续租 |
| **Rebind** | 客户端→服务器 | 重新绑定 |
| **Release** | 客户端→服务器 | 释放地址 |
| **Information-request** | 客户端→服务器 | 仅获取配置 (无地址) |
| **Reconfigure** | 服务器→客户端 | 服务器主动通知更新 |

## 🔧 关键概念

### DUID (DHCP 唯一标识符)
每个 DHCP 参与者有唯一的 DUID，类型包括：
- **DUID-LLT**: 基于链路层地址 + 时间
- **DUID-EN**: 基于厂商企业编号
- **DUID-LL**: 基于链路层地址
- **DUID-UUID**: 基于 UUID

### IA (身份关联)
- **IA_NA**: 非临时地址分配
- **IA_PD**: 前缀委派
- ~~IA_TA~~: 临时地址 (已废弃)

### 租约时间
- **T1**: 首选生存时间 (默认 0.5 × 租期)
- **T2**: 有效生存时间 (默认 0.8 × 租期)

---

## 🔒 安全考虑

1. **客户端**: 可能收到恶意服务器配置
2. **服务器**: 可能遭受 DoS 攻击
3. **缓解措施**:
    - DHCPv6-Shield (RFC 7610)
    - 认证选项 (RKAP)
    - 源地址验证 (SAVI)

---

## 📦 常用选项

| 选项代码 | 名称 | 用途 |
|---------|------|------|
| 1 | Client ID | 客户端标识 |
| 2 | Server ID | 服务器标识 |
| 3 | IA_NA | 非临时地址 |
| 6 | IA_PD | 前缀委派 |
| 23 | DNS 服务器 | DNS 配置 |
| 24 | 域名搜索列表 | DNS 搜索域 |
| 14 | Rapid Commit | 快速提交 (2 次消息完成) |

---

## 🌐 网络端口

- **客户端**: UDP 546
- **服务器/中继**: UDP 547
- **组播地址**: FF02::1:2 (所有 DHCP 服务器/中继)

---

## 📝 总结

RFC 9915 是 DHCPv6 的最新标准，主要做了**简化和澄清**：
- 移除了未被广泛使用的复杂功能
- 整合了多年来的勘误
- 保持了与现有实现的兼容性

对于网络工程师来说，**核心操作没有变化**，只是更清晰、更简洁了。
