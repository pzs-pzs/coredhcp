# CoreDHCP 网络拓扑示例

## 基本拓扑 - 直连场景

```
                    +----------------------+
                    |                      |
    +-----------+   |   CoreDHCP Server    |
    |   Client  |---|   192.168.1.10       |
    +-----------+   |   Port: 67/547       |
                    +----------------------+
                           |
                    (同一网段)
```

**配置示例:**
```yaml
server4:
    listen: "192.168.1.10:67"
    plugins:
        - range: "leases.db 192.168.1.100 192.168.1.200 3600s"
```

## 中继拓扑 - 单一网关

```
    +-----------+       +-------------+       +----------------------+
    |           |       |             |       |                      |
    |  Client   |-------|  Router     |-------|   CoreDHCP Server    |
    | 192.168.1 |       |  Relay      |       |   10.0.0.2           |
    +-----------+       | 192.168.1.1 |       |   Port: 547          |
                        +-------------+       +----------------------+
                              |                        |
                         (LAN侧)                  (WAN侧)
```

**数据流向:**
1. Client → Router: DHCP Solicit (广播)
2. Router → CoreDHCP: Relay-Forward (单播, giaddr=192.168.1.1)
3. CoreDHCP → Router: Relay-Reply (单播)
4. Router → Client: DHCP Reply (广播)

**路由器配置 (OpenWrt):**
```bash
# /etc/config/dhcp
config dhcp 'relay'
    option interface 'lan'
    option relay_server '10.0.0.2'
```

**CoreDHCP 配置:**
```yaml
server6:
    listen: "[10.0.0.2]:547"
    plugins:
        - server_id: LL 00:de:ad:be:ef:00
        - classify: "/etc/coredhcp/classes.yaml"
        - file: "leases6.txt"
```

## 中继拓扑 - 多分支

```
                                   +----------------------+
    +-----------+       +---------|                       |
    |           |       |         |   CoreDHCP Server     |
    | Client-1  |-------|Router-1 |   (总部)              |
    +-----------+       +---------+   10.0.0.2            |
    192.168.1.x         |           +----------------------+
                        |
                    (WAN)
                        |
    +-----------+       +---------+
    |           |       |         |
    | Client-2  |-------|Router-2 |
    +-----------+       +---------+
    192.168.2.x         |
                        |
                    (WAN)
                        |
    +-----------+       +---------+
    |           |       |         |
    | Client-3  |-------|Router-3 |
    +-----------+       +---------+
    192.168.3.x
```

**特点:**
- 多个分支机构通过各自的 WAN 连接到总部
- 所有 DHCP 请求集中到总部 CoreDHCP 服务器处理
- 可根据客户端特征（如 MAC OUI）区分不同分支

**配置示例:**

```yaml
# /etc/coredhcp/classes.yaml
classes:
  - name: "branch1"
    conditions:
      mac_prefix:
        - "00:11:22"  # 分支1设备OUI

  - name: "branch2"
    conditions:
      mac_prefix:
        - "33:44:55"  # 分支2设备OUI

  - name: "branch3"
    conditions:
      mac_prefix:
        - "66:77:88"  # 分支3设备OUI
```

```bash
# leases6.txt - 为不同分支分配不同网段
# 分支1 - 2001:db8:1::/64
00:11:22:33:44:55:66:77: 2001:db8:1::100

# 分支2 - 2001:db8:2::/64
33:44:55:66:77:88:99:aa: 2001:db8:2::100

# 分支3 - 2001:db8:3::/64
66:77:88:99:aa:bb:cc:dd: 2001:db8:3::100
```

## 双栈 DHCPv4/DHCPv6 中继

```
                        +-------------+
    +-----------+       |             |       +----------------------+
    |           |       |  Router     |       |                      |
    |   Client  |-------|  (Relay)    |-------|   CoreDHCP Server    |
    |   Dual    |-------|  giaddr:    |-------|   DHCPv4: 67         |
    |   Stack   |       |  192.168.1.1|       |   DHCPv6: 547        |
    +-----------+       +-------------+       +----------------------+
```

**CoreDHCP 双栈配置:**

```yaml
# 同时启用 DHCPv4 和 DHCPv6
server4:
    listen: "0.0.0.0:67"
    plugins:
        - classify: "/etc/coredhcp/classes.yaml"
        - range: "leases4.db 192.168.1.100 192.168.1.200 3600s"

server6:
    listen: "[::]:547"
    plugins:
        - server_id: LL 00:de:ad:be:ef:00
        - classify: "/etc/coredhcp/classes.yaml"
        - range: "leases6.db 2001:db8::1 2001:db8::1000 3600s"
```

## 冗余部署 - 主备服务器

```
                        +-------------+
                        |             |-----> +------------------+
                        |  Router     |       | CoreDHCP Master  |
    +-----------+       |  (Relay)    |       | 10.0.0.2 (主)    |
    |           |       |             |       +------------------+
    |  Client   |-------|             |
    |           |       |             |-----> +------------------+
    +-----------+       +-------------+       | CoreDHCP Backup   |
                                                | 10.0.0.3 (备)    |
                                                +------------------+
```

**实现方式:**
1. 使用 keepalived 或 VRRP 实现虚拟 IP 漂移
2. 或在路由器配置多个中继服务器地址

**路由器配置多个中继服务器:**
```bash
# OpenWrt 支持多个 relay_server
config dhcp 'relay'
    option relay_server '10.0.0.2 10.0.0.3'
```

## 多级中继

```
+-----------+       +-------------+       +-------------+       +--------------+
|           |       |             |       |             |       |              |
|  Client   |-------|  L2 Switch  |-------|  L1 Router |-------|  CoreDHCP    |
|           |       |  (Relay)    |       |  (Relay)    |       |  Server      |
+-----------+       +-------------+       +-------------+       +--------------+
                            |                        |
                        (二级中继)                (一级中继)
```

**注意事项:**
- 中继跳数 (Hop Count) 会累积
- 每级中继会添加自己的 Relay Message 封装
- CoreDHCP 需要解析多层封装获取原始客户端信息

## VLAN 场景

```
+----------------------------------------------------------+
|                                                         |
|  +--------+      +--------+      +--------+             |
|  | VLAN10 |      | VLAN20 |      | VLAN30 |             |
|  | 192.168|      | 192.168|      | 192.168|             |
|  | .1.x   |      | .2.x   |      | .3.x   |             |
|  +---+----+      +---+----+      +---+----+             |
|      |              |              |                    |
|      +--------------+--------------+                    |
|                     |                                   |
|              +------+------+                           |
|              |  L3 Switch   |                           |
|              |  (Relay)      |                           |
|              +-------+------+                           |
|                      |                                   |
|              +-------+-------+                           |
|              |  CoreDHCP     |                           |
|              |  10.0.0.2     |                           |
|              +---------------+                           |
+----------------------------------------------------------+
```

**按 VLAN 分类客户端:**

```yaml
classes:
  - name: "vlan10-devices"
    conditions:
      mac_prefix:
        - "00:11:22"  # VLAN10 设备

  - name: "vlan20-devices"
    conditions:
      mac_prefix:
        - "33:44:55"  # VLAN20 设备
```

## 网络拓扑选择建议

| 场景 | 推荐拓扑 | 配置复杂度 | 适用规模 |
|------|----------|-----------|---------|
| 小型办公室 | 直连 | 低 | <50 设备 |
| 单一大楼 | 单一网关中继 | 中 | 50-500 设备 |
| 多分支企业 | 多分支中继 | 高 | >500 设备 |
| 高可用需求 | 主备冗余 | 高 | 关键业务 |
| 多部门网络 | VLAN 中继 | 中 | 中大型企业 |
